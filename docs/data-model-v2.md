# V2 Data Model & Firestore Schema

This document details the database layout and document structures for version 2.0 schemas.

---

## 1. Firestore Database Layout
All V2 data is stored in dedicated, version-segregated Firestore root collections. 

```
Firestore Database
 ├── assessmentsV2/           # Keyed by user uid
 ├── labReportsV2/            # Keyed by reportId
 ├── recommendationsV2/       # Keyed by assessmentId
 └── regionalContextV2/       # Keyed by user uid
```

---

## 2. Document Schemas

### 2.1 Collection: `assessmentsV2`
Holds complete clinical risk surveys for individual users.
- **Document ID**: User UID (`uid`)
- **Fields**:
  - `userId`: String (User UID)
  - `age`: Number
  - `gender`: String ("male" | "female" | "other")
  - `heightCm`: Number
  - `weightKg`: Number
  - `smoking`: String ("never" | "former" | "current")
  - `exercise`: String ("none" | "light" | "moderate" | "active")
  - `familyHistory`: String
  - `symptoms`: String
  - `alcohol`: String ("never" | "occasional" | "heavy")
  - `sleepHours`: Number
  - `systolicBP`: Number
  - `diastolicBP`: Number
  - `heartRate`: Number
  - `fastingBloodSugar`: Number
  - `schemaVersion`: String ("2.0")
  - `updatedAt`: String (ISO Datetime)

---

### 2.2 Collection: `labReportsV2`
Biomarkers parsed from clinical lab tests.
- **Document ID**: Unique Report ID (`reportId`)
- **Fields**:
  - `reportId`: String
  - `userId`: String
  - `uploadDate`: String (ISO Datetime)
  - `bloodSugarHbA1c`: Number (optional)
  - `totalCholesterol`: Number (optional)
  - `hdlCholesterol`: Number (optional)
  - `ldlCholesterol`: Number (optional)
  - `triglycerides`: Number (optional)
  - `vitaminD3`: Number (optional)
  - `vitaminB12`: Number (optional)
  - `thyroidTSH`: Number (optional)
  - `isVerified`: Boolean
  - `verifiedBy`: String (optional)
  - `schemaVersion`: String ("2.0")
  - `updatedAt`: String (ISO Datetime)

---

### 2.3 Collection: `recommendationsV2`
Personalized diet, fitness, and lifestyle advice generated by ML/AI.
- **Document ID**: Assessment ID
- **Fields**:
  - `assessmentId`: String
  - `userId`: String
  - `riskCategory`: String ("low" | "moderate" | "high")
  - `diseaseSpecificRisks`: Map:
    - `diabetesRiskPercent`: Number
    - `heartDiseaseRiskPercent`: Number
    - `hypertensionRiskPercent`: Number
  - `dietPlanRegional`: Map:
    - `cuisinePreference`: String
    - `dailyMeals`: Array of String
    - `keyExclusions`: Array of String
  - `fitnessRoutine`: Map:
    - `exercises`: Array of String
    - `frequencyWeekly`: Number
    - `durationMinutesPerSession`: Number
  - `clinicalReferralNeeded`: Boolean
  - `physicianGuidanceNotes`: String
  - `schemaVersion`: String ("2.0")
  - `updatedAt`: String (ISO Datetime)

---

### 2.4 Collection: `regionalContextV2`
Localization parameters.
- **Document ID**: User UID (`uid`)
- **Fields**:
  - `userId`: String
  - `language`: String ("en" | "hi" | "gu")
  - `preferredDietaryType`: String ("vegetarian" | "non-vegetarian" | "vegan")
  - `stateOrRegionCode`: String
  - `customRegionalRules`: Array of String
  - `schemaVersion`: String ("2.0")
  - `updatedAt`: String (ISO Datetime)
